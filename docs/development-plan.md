# 忆伴相册开发计划

更新日期：2026-05-22

## 当前阶段

当前项目处于双端 MVP 演示阶段。

目标不是上线产品，而是先用两个网页跑通核心亲情闭环：

```text
子女手机端上传照片/发送留言
-> 相册端展示照片和留言
-> 老人进入面对面陪伴并回复
-> 系统生成亲情摘要和家庭记忆
-> 子女端查看并回应
```

## 今日完成

### 1. 工程与服务

- 新增零依赖 Node 本地演示服务 `server.js`
- 新增 `package.json`
- 提供本地入口：
  - 子女手机端：`http://localhost:4173/`
  - 相册端：`http://localhost:4173/device/`
- 新增本地 JSON 持久化：
  - 数据文件：`data/demo-state.json`
  - `data/` 已被 `.gitignore` 忽略
- 新增演示数据重置接口：
  - `POST /api/dev/reset`

### 2. 子女手机端

- 改为固定 App 式布局
- 底部功能栏：
  - 首页
  - 留言
  - 照片
  - 对话
  - 记忆
- 支持发送留言
- 支持查看妈妈回复
- 支持添加照片
- 支持本地图片选择、预览和提交
- 支持照片列表展示和删除
- 支持生成亲情摘要
- 支持一键发送建议回应
- 支持查看最近对话
- 支持查看和删除家庭记忆
- 增加操作反馈 toast
- 增加重置演示按钮

### 3. 相册端

- 大屏相册界面
- 家庭照片轮播
- 时间、日期、状态展示
- 面对面陪伴模式模拟
- 麦克风开关状态
- 麦克风关闭时阻止输入
- 未进入面对面模式时阻止输入
- 家人留言展示
- 新留言/已读状态
- 进入面对面模式时标记留言已读
- 支持围绕当前照片模拟老人说话
- 支持针对最新留言回复家人

### 4. 核心业务能力

- 照片：
  - 添加照片
  - 删除照片
  - 自定义图片 data URL 演示
  - 相册至少保留一张照片
- 留言：
  - 子女发送留言
  - 相册端已读
  - 老人回复留言
  - 子女端查看回复
- 对话：
  - 面对面陪伴模式下创建对话
  - 基于当前照片生成陪伴回复
- 摘要：
  - 根据最近可分享对话生成亲情摘要
  - 生成建议回复
- 记忆：
  - 生成摘要时沉淀家庭记忆
  - 家庭记忆可删除
  - 再次围绕同一照片对话时引用相关记忆
- 隐私：
  - 识别“别发”“别告诉孩子”“只记着”“删掉”“别听了”“休息吧”
  - 隐私片段不进入摘要
  - local_only 回复不展示给子女
  - 敏感关键词片段不进入摘要

## 已验证

- `node --check server.js` 通过
- 手机端 `/` 返回 200
- 相册端 `/device/` 返回 200
- 子女发留言 -> 老人回复 -> 子女端查看回复 跑通
- 生成摘要 -> 沉淀记忆 -> 再次对话引用记忆 跑通
- 添加自定义照片 -> 相册显示 -> 删除照片 跑通
- 隐私指令“这段别发给孩子”不会进入摘要
- 服务重启后数据可持久化
- 重置接口可恢复默认演示数据

## 下一步计划

### P0：继续完善 MVP 闭环

1. 梳理 API 结构
   - 当前 `server.js` 已变大
   - 下一步拆分为 `services`、`models`、`routes` 或迁移到正式 FastAPI

2. 完善消息状态
   - unread
   - heard
   - replied
   - archived

3. 完善回复确认流程
   - 当前回复直接发送
   - 后续应增加“我这样发可以吗？”确认状态

4. 完善家庭记忆
   - 支持编辑记忆标题和内容
   - 支持标记不准确
   - 支持按照片查看相关记忆

### P1：真实能力接入

1. 真实图片上传
   - 当前使用 data URL 演示
   - 后续改为文件上传和本地/对象存储

2. AI 图片理解
   - 自动生成照片描述
   - 自动生成场景标签
   - 自动生成回忆引导问题

3. 真实语音链路
   - 录音
   - ASR
   - TTS
   - 打断

4. AI Gateway
   - 抽象模型调用
   - 避免业务代码直接绑定模型厂商

### P2：产品化能力

1. 家庭账号
2. 设备绑定
3. 多家庭成员
4. 权限控制
5. 隐私审计
6. 部署脚本
7. 自动化测试

后续重点应该从“继续堆页面”转为：

1. 收敛代码结构
2. 接入真实 AI/语音/图片能力
