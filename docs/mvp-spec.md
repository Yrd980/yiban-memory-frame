# 忆伴相册 MVP 功能规格

版本：0.1  
阶段：双端演示 / MVP 定义  
目标：用手机端和相册端跑通第一条亲情连接闭环。

## 1. MVP 目标

忆伴相册第一版 MVP 不追求完整硬件、真实语音和复杂 AI 能力，而是验证两个核心假设：

1. 老人是否愿意围绕家庭照片和相册自然表达。
2. 子女是否会因为亲情摘要而产生真实回应。

第一版闭环：

```text
子女上传照片/留言
-> 相册端展示
-> 老人面对相册说话
-> 系统整理回复和摘要
-> 子女端看到摘要并回应
-> 相册端展示子女回应
```

## 2. 端侧定位

### 2.1 子女手机端

用途：

- 上传照片
- 发送留言
- 查看亲情摘要
- 快速回复老人
- 查看最近对话
- 后续查看家庭记忆

设计原则：

- 移动端 App 式体验
- 底部功能栏切换
- 不做复杂后台
- 不展示监控感数据

### 2.2 相册端

用途：

- 全屏轮播家庭照片
- 展示家人留言
- 进入面对面陪伴模式
- 模拟老人语音输入
- 展示 AI 陪伴回复
- 显示麦克风和陪伴状态

设计原则：

- 远距离可读
- 状态明确
- 不像监控设备
- 照片是主角，AI 退到背后

## 3. 功能一：照片上传与相册展示

### 3.1 目标

子女上传一张家庭照片，并添加说明、人物、场景和回忆引导。照片进入相册轮播，并成为 AI 对话的上下文。

### 3.2 子女端流程

1. 进入 `照片`
2. 添加照片
3. 填写标题、说明、人物、场景、回忆引导
4. 提交
5. 照片进入相册端轮播队列

### 3.3 相册端流程

1. 拉取照片列表
2. 默认轮播照片
3. 面对面模式下可围绕当前照片接话

### 3.4 MVP 字段

```json
{
  "id": "photo_001",
  "family_id": "family_001",
  "uploader_id": "user_child_001",
  "image_url": "/uploads/photo_001.jpg",
  "title": "小宝在公园放风筝",
  "caption": "小宝今天去公园放风筝了，想给妈妈看看",
  "people_tags": ["小宝"],
  "scene_tags": ["公园", "放风筝"],
  "time_hint": "今天下午",
  "memory_prompt": "你以前带孩子放过风筝吗？",
  "status": "ready",
  "created_at": "2026-05-22T16:00:00+08:00"
}
```

### 3.5 MVP 规则

- 先允许表单方式添加照片卡片。
- 后续再接真实图片上传。
- 子女填写的信息优先于 AI 推测。
- AI 不确认人物身份，只能引用已标注内容。

## 4. 功能二：子女留言与老人回复

### 4.1 目标

子女可以发送留言；老人端展示或朗读留言；老人自然回复；系统整理后发回子女端。

### 4.2 子女端流程

1. 进入 `留言`
2. 输入文字
3. 发送到相册端
4. 等待老人回复

### 4.3 相册端流程

1. 显示最新留言
2. 老人进入面对面模式后可回复
3. 系统整理老人原话
4. 发送到子女端

### 4.4 数据模型

留言：

```json
{
  "id": "msg_001",
  "family_id": "family_001",
  "sender_id": "user_child_001",
  "type": "text",
  "content": "妈，小宝今天去公园放风筝了，给你看看。",
  "related_photo_id": "photo_001",
  "status": "unread",
  "created_at": "2026-05-22T16:20:00+08:00",
  "heard_at": null,
  "replied_at": null
}
```

老人回复：

```json
{
  "id": "reply_001",
  "message_id": "msg_001",
  "raw_text": "小宝又长高了，像他妈妈小时候。周末想吃奶奶包的饺子就回来。",
  "polished_text": "奶奶说小宝又长高了，像妈妈小时候。她问小宝周末想不想回来吃她包的饺子。",
  "status": "pending_confirm",
  "share_policy": "send_to_sender",
  "created_at": "2026-05-22T16:25:00+08:00"
}
```

### 4.5 MVP 规则

- 第一版只做文字留言。
- 老人回复用文本输入模拟语音。
- AI 整理回复时不能添加老人没说过的承诺。
- 子女端展示整理后的回复，不展示系统痕迹。

## 5. 功能三：面对面陪伴模式

### 5.1 目标

老人坐到相册前后，相册进入短时间、可见、可退出的陪伴状态。老人可以直接说话，不必每句都喊唤醒词。

### 5.2 MVP 触发

相册端按钮模拟：

```text
坐到相册前
```

点击后进入：

```text
FACE_TO_FACE_READY
```

### 5.3 状态机

```text
IDLE_ALBUM
  -> PRESENCE_DETECTED
  -> FACE_TO_FACE_READY
  -> LISTENING
  -> THINKING
  -> SPEAKING
  -> FACE_TO_FACE_READY
  -> IDLE_ALBUM
```

### 5.4 会话模型

```json
{
  "id": "session_001",
  "family_id": "family_001",
  "device_id": "device_001",
  "elder_id": "elder_001",
  "started_at": "2026-05-22T16:40:00+08:00",
  "ended_at": null,
  "trigger": "presence",
  "current_photo_id": "photo_001",
  "privacy_mode": "normal",
  "summary_allowed": true
}
```

### 5.5 MVP 规则

- 相册模式下照片自动轮播。
- 面对面模式下照片暂停，围绕当前照片聊天。
- 老人说“休息吧”“别听了”时退出。
- 麦克风关闭后不接受输入。

## 6. 功能四：AI 亲情摘要

### 6.1 目标

把老人和相册的日常对话整理成子女愿意读、读得懂、容易回应的亲情摘要。

### 6.2 摘要不是

- 监控报告
- 原始聊天全文
- 情绪评分报表
- 医疗或风险诊断

### 6.3 摘要应该是

```text
妈妈今天看到小宝放风筝的照片很开心，说小宝跑起来的样子像你小时候。
她还提到周末想包饺子，问小宝想不想回来吃。
```

### 6.4 数据模型

```json
{
  "id": "summary_001",
  "family_id": "family_001",
  "elder_id": "elder_001",
  "recipient_ids": ["user_child_001"],
  "type": "daily_memory",
  "title": "妈妈今天看到小宝的照片很开心",
  "body": "妈妈今天看到小宝放风筝的照片，说小宝长高了，跑起来的样子像你小时候。",
  "suggested_replies": [
    "妈，小宝看到你这么说一定很开心。周末我们看看时间回去吃饺子。"
  ],
  "source_turn_ids": ["turn_001"],
  "source_photo_ids": ["photo_001"],
  "sensitivity": "normal",
  "share_status": "ready",
  "created_at": "2026-05-22T17:00:00+08:00"
}
```

### 6.5 MVP 规则

- 子女端可手动生成摘要。
- 会话结束后可自动生成摘要。
- 摘要必须过滤隐私片段。
- 摘要提供建议回复。
- 子女从摘要到回复不超过两步。

## 7. 功能五：家庭记忆库

### 7.1 目标

把照片、老人讲述、子女补充和 AI 摘要沉淀成可长期使用的家庭记忆。

### 7.2 记忆类型

```text
photo_memory
person_note
place_memory
family_story
preference
```

### 7.3 数据模型

```json
{
  "id": "memory_001",
  "family_id": "family_001",
  "elder_id": "elder_001",
  "type": "photo_memory",
  "title": "小宝放风筝让妈妈想起女儿小时候",
  "content": "妈妈说小宝放风筝的样子像女儿小时候，也提到周末想包饺子给小宝吃。",
  "people": ["妈妈", "女儿", "小宝"],
  "places": ["公园"],
  "related_photo_ids": ["photo_001"],
  "source_turn_ids": ["turn_001", "turn_002"],
  "source_summary_id": "summary_001",
  "confidence": "confirmed_by_elder",
  "share_policy": "family_visible",
  "sensitivity": "normal",
  "created_at": "2026-05-22T17:30:00+08:00"
}
```

### 7.4 置信度

```text
confirmed_by_family
confirmed_by_elder
inferred_by_ai
inferred_from_repeated_mentions
```

### 7.5 MVP 规则

- 第一版从摘要生成记忆。
- 子女端后续展示家庭记忆列表。
- 记忆必须保留来源。
- AI 引用记忆时要区分确认事实和推测。

## 8. 功能六：隐私与安全控制

### 8.1 目标

让老人和子女都清楚：忆伴相册不是监控设备。它不偷拍、不远程监听、不默认公开原始对话。

### 8.2 产品边界

必须做到：

- 无摄像头
- 麦克风状态可见
- 家属不能远程监听
- 默认发摘要，不发全文
- 老人隐私指令优先

### 8.3 隐私指令

```text
这段别发
别告诉孩子
只记着就行
不要发了
删掉刚才那段
别听了
休息吧
```

### 8.4 分享策略

```text
family_visible
summary_only
local_only
private
deleted
```

### 8.5 对话片段字段

```json
{
  "id": "turn_001",
  "speaker": "elder",
  "text": "这段别告诉孩子。",
  "privacy_flag": "local_only",
  "summary_allowed": false,
  "memory_allowed": true,
  "deleted": false
}
```

### 8.6 MVP 规则

- 相册端提供麦克风开关。
- “别发/别告诉孩子/只记着/删掉/别听了”必须优先识别。
- 摘要生成过滤隐私片段。
- 子女端不展示原始全文。
- 子女端不提供远程监听入口。

## 9. MVP 暂不做

- 真实硬件传感器
- 真实 ASR/TTS
- 真实照片 AI 识别
- 摄像头
- 视频通话
- 医疗诊断
- 情绪评分报表
- 多家庭复杂权限
- 向量检索和知识图谱
- 完整云端部署

## 10. 第一版验收标准

1. 子女端可以添加照片和留言。
2. 相册端可以轮播照片并展示留言。
3. 相册端可以进入面对面陪伴模式。
4. 老人围绕当前照片输入一句话后，系统能生成陪伴回复。
5. 子女端可以生成亲情摘要。
6. 子女端可以根据摘要快速回复。
7. 老人说“这段别发”后，摘要不会包含该内容。
8. 麦克风关闭后，相册端不接受输入。
9. 整体体验不呈现监控感。
10. 两个网页能演示完整亲情闭环。
